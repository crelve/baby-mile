# 要件定義 — Baby Mile

> 作成: 2026-05-04 / Phase 6
> 担当: 本田 圭佑 + 今田 美桜
> 形式: ハイブリッド（EARS + 画面仕様）

---

## Part 1: ビジネス要件

- BR1: 0-3歳の子供を持つ親が、マイルストーンを50+のプリセットから選び、写真+日付+メモで記録できる
- BR2: 月齢別発達チェックリストで、達成済み/未達成を一覧確認できる
- BR3: マイルストーンカードを自動生成し、SNSへシェアできる
- BR4: 月額¥480の有料プランで、全機能をアンロックできる
- BR5: 7日間の無料トライアルから課金へ転換できる
- BR6: 39言語で利用できる

## Part 2: 機能要件（EARS）

### マイルストーン記録
- F1: ユーザがマイルストーンを記録するとき、システムはプリセット50+から選択させ、写真+日付+メモを保存しなければならない
- F2: 写真を選択するとき、システムはカメラ撮影またはフォトライブラリ選択を提供しなければならない
- F3: マイルストーンを記録したとき、システムは月齢を自動算出して保存しなければならない
- F4: 無料ユーザがマイルストーンを11件目記録しようとするとき、システムはペイウォールを表示しなければならない

### 発達チェックリスト
- F5: ユーザが月齢を選んだとき、システムは該当月齢の発達目安一覧を表示しなければならない
- F6: ユーザが達成チェックを入れたとき、システムはマイルストーンレコードを自動生成しなければならない

### マイルストーンカード
- F7: ユーザがカード生成を要求したとき、システムは縦長/正方形/横長の3サイズを切替提供しなければならない
- F8: 無料ユーザが4K書き出しを要求したとき、システムはペイウォールを表示しなければならない

### タイムライン
- F9: ユーザがタイムライン画面を開いたとき、システムは全マイルストーンを時系列降順でリスト表示しなければならない

### 通知
- F10: 子供の月齢が新しい節目に到達したとき、システムは「○ヶ月のマイルストーンチェック」をローカル通知しなければならない（M3）

### オンボーディング
- F11: 初回起動のとき、システムは3ステップで「子供の生年月日 → 月齢ピックアップ → 1件目のマイルストーン記録」へ誘導しなければならない（M4）

### 課金
- F12: ユーザがプレミアム購入したとき、システムは RevenueCat 経由で entitlement を有効化しなければならない
- F13: トライアル期間中、システムは AdMob 広告を非表示にしなければならない

## Part 3: 非機能要件

| ID | 項目 | 内容 |
|----|------|------|
| NF1 | パフォーマンス | アプリ起動 2秒以内、カード生成 1秒以内 |
| NF2 | オフライン | ネット未接続でも記録可能、復帰後同期 |
| NF3 | 多言語 | 39言語フルサポート |
| NF4 | アクセシビリティ | VoiceOver / TalkBack 対応 |
| NF5 | プライバシー | Privacy Policy 必須、写真は端末保存優先 |
| NF6 | セキュリティ | Firestore/Storage Rules で uid 限定 |
